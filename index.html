Algorithm Design

The scheduling engine maintains a ready queue, executes each process for the current quantum, admits newly arrived processes, and recomputes quantum from remaining burst times.

Scheduling Engine

Maintains pending, ready, running, and completed process states. It records every execution segment for the Gantt chart.

Data & Metrics

Validates arrivals and bursts, tracks completion time, and calculates waiting, turnaround, response time, context switches, and CPU utilization.

Visualization

Transforms timeline segments into an interactive Gantt chart, quantum log, result table, and comparison bars.

Quantum Strategies selectable in simulator
Average: Q = ceil(sum(remaining burst times) / number of ready processes)
Median: Q = ceil(median(remaining burst times in ready queue))
Load-aware: Q = ceil((average + median) / 2), then reduce slightly when queue length is high

Sort and Admit

Processes are sorted by arrival time. At each CPU time, all processes whose arrival time has passed are moved into the ready queue.

Calculate Quantum

The scheduler computes a fresh time slice from the ready queue before selecting the next process.

Execute or Preempt

The selected process runs for min(quantum, remaining burst). Finished processes leave the queue; unfinished ones return to the end.

Compute Metrics

Completion time gives turnaround time. Waiting time is turnaround minus burst. Response time is first start minus arrival.
